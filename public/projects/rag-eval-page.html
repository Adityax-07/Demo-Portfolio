Strategies Compared
8 retrieval architectures, same benchmark.
Each system retrieves differently — the same 50-question eval exposes exactly what breaks and why.
S1 · Baseline
Base LLM
No retrieval · parametric only
Answers from training weights alone. The floor every other system must beat — surprisingly competitive on coherence.
Score 0.75
S2 · Naive
Naive RAG
FAISS top-3 semantic search
Classic RAG: embed the question, find nearest 3 chunks, answer from those. Good baseline; misses exact-term queries.
Score 0.74
S3 · Hybrid
Hybrid RAG
BM25 + FAISS merged via RRF
Keyword search meets semantic search. Catches acronyms and technical terms FAISS alone misses.
Score 0.72
S4 · Reranking
Reranking RAG
FAISS × 10 → cross-encoder top-3
Fetch 10 candidates, cross-encoder re-scores each pair together. Better precision — LLM sees only the most relevant chunks.
Score 0.75
S5 · HyDE
HyDE RAG
Hypothetical answer embedding
Generate a fake answer first, embed that instead of the question. Bridges the query/document embedding gap.
Score 0.73
S6 · Rewriting
Query Rewriting
3 rewrites · FAISS × 3 → RRF
Rewrite the question 3 ways, run FAISS for each, merge via RRF. Better recall for ambiguous queries.
Score 0.72
S7 · Advanced
Advanced RAG
Rewriting + Hybrid + Reranking
All three improvements stacked: multi-query generation, BM25+FAISS hybrid, cross-encoder reranking. Best faithfulness among fixed pipelines.
Score 0.77 · Best Fixed
S8 · Adaptive ✨
Adaptive RAG
Dynamic routing per query
Routes each query to the right pipeline: Simple → Naive · Complex → Advanced · Ambiguous → Hybrid. Avoids noisy over-retrieval for easy questions.
Faithfulness 1.000 ✨

Key Insights
What the numbers reveal
✨
Perfect faithfulness
Adaptive RAG hits 1.000 faithfulness by routing simple queries to Naive RAG — avoiding over-retrieval that introduces noisy context.
🧭
Routing > stacking
Advanced RAG stacks everything and scores 0.77. Adaptive RAG routes intelligently — hitting the same tier without burning compute on every query.
📉
Hallucination floor
Advanced RAG achieves 0.18 hallucination rate — 4× lower than base LLM's 0.78. Grounded retrieval is the single biggest lever.
🎯
Reranking wins precision
Cross-encoder reads question and chunk together — bi-encoders encode separately. 0.82 faithfulness vs Naive's 0.77.
🔀
Hybrid recall advantage
BM25 catches exact terms FAISS misses. RRF merges both lists without trade-offs. Critical for technical acronyms and precise domain terms.
🪄
HyDE bridges the gap
Question and answer embeddings live in different spaces. Generating a hypothetical answer first puts the query vector into answer-space.

Get Started
Run the full benchmark in 3 steps
bash · zsh · powershell
01 — clone & install
git clone https://github.com/Adityax-07/RAG-Systems-eval-suite.git
cd llm-eval-framework && pip install -e ".[dev]"
02 — add Groq API key (free at console.groq.com)
cp .env.example .env && echo "GROQ_API_KEY=gsk_..." >> .env
03 — run the benchmark
python examples/compare_systems.py
# 8 systems × 10 questions × 10 judges — results saved to SQLite
