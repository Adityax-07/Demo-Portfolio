How It Works
Three architectures. One benchmark.
Same question, same moment, real output from all three.
System 1 · Baseline
Baseline LLM
Llama-3.1-8B · Groq · zero-shot
  • Zero setup, instant
  • Broad topic coverage
  • No infrastructure
  • Hallucinates 43.2%
  • No domain knowledge
System 2 · RAG
RAG Pipeline
FAISS + all-MiniLM-L6-v2 · top-3
  • Update docs, no retrain
  • Grounded + traceable
  • Citations visible
  • Retrieval lag ~2.1s
  • Chunk quality matters
System 3 · Fine-Tuned
Fine-Tuned Model
Qwen2.5-1.5B + LoRA r=8 α=32
  • 10× cheaper per query
  • 0% hallucination
  • 3× faster · 0.4s
  • Hard to update
  • Fixed domain

Get Started
Run locally in 3 steps
bash · zsh · powershell
01 — clone & install
git clone https://github.com/Adityax-07/LLM-vs-RAG-vs-Fine-Tuning-.git
cd LLM-vs-RAG-vs-Fine-Tuning- && pip install -r requirements.txt
02 — add Groq API key (free at console.groq.com)
echo "GROQ_API_KEY=gsk_..." > .env
03 — launch
streamlit run demo.py
